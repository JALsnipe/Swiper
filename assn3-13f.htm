Assignment 3: A Gestural Video Player

 

Your third assignment is designed to make you feel more comfortable with HTML5 and JavaScript, while giving you a chance to demonstrate what you have learned about user interface design.

 

You will be designing, building, and documenting, by yourself, an application that uses one of the gesture recognizers that is referenced on the webpage for the $1 unistroke gesture recognizer. Your application will be a video player whose functionality is provided by the HTML5 <video> element. User interaction with your player should be done entirely by gestures (i.e., without the need for buttons, sliders, keyboards, or the controls attribute). You should support play, pause, seek (i.e., setting the currentTime attribute), and control of the playbackRate, volume, muted, width, and height attributes. Interaction with your video player should be performed using gestures whose templates you define yourself (instead of using the set of templates defined in the original source code for the recognizer you have chosen). Note that you are welcome to treat the template set you define as complete (i.e., you do not have to support creation of templates by your user). You are welcome to hardcode a single video into your application and you do not need to save the results of using your application (i.e., your application can start up the same way each time, and does not need to delete or modify anything in your file system).

 

Each gesture recognizer's web page has a link to the JavaScript code for the recognizer and a live demo. Thus, you not need to code the recognition algorithm yourself, and the JavaScript demo embedded in each recognizer's web page provides an example of how to use the HTML5 <canvas> element to create an input area. You can find a "cheat sheet" comparing the different gesture recognizers in Table 2 of the ICMI 2012 paper on the $P recognizer.

 

While we have provided you with a set of functional requirements for this assignment, a major component of the grading process will be our heuristic analysis of your design, and implemented application, focusing on usability. 

 

We will grade your assignment in the current version of Chrome. To avoid Browser incompatibility issues, please make sure that you develop and test your assignment in Chrome, using at least version 30. (As minor examples of browser-specific differences, in the top-level demo code on the recognizer web pages, zooming, resizing, or scrolling the page causes strokes to be drawn at an offset in Chrome 30 under Windows, while only zooming and resizing cause this problem in IE 10 under Windows. Audio playback at speeds other 1.0 is significantly distorted in IE 10 as compared with Chrome 30. Since there may be other more significant differences that will affect how your application performs. please use Chrome.)

Design

 

You should begin by playing with the recognizer you choose to get an idea of the kind of performance you might expect from it. Do the gestures you want to use feel right? Can you get the recognizer to perform well with them? Then, start thinking about possible ideas and trying them out by using some of the lo-fi prototyping techniques discussed in class and used in Assignment 2. In other words, you should sketch out and explore your ideas before you begin coding!

 

Do try to make it as easy as you can for a computer-knowledgeable user who has never seen your application before to be able to determine quickly how to use it, even if you accomplish that by referring your user to online documentation when they first run your application.

 

Documentation

 

As part of your submission, you should include a pdf file with embedded images that

  • Documents your design and explains how it developed, using images of your initial sketches or paper prototypes.

  • Explains what your application does and how to use it, using screenshots.

  • Justifies your user interface design decisions.

Please pay special attention to that last bullet item!

Hints

 

You can learn about the <video> element and its API by trying a demonstration (be sure to look at its source code) and reading a description.

 

You should use the JavaScript code for the recognizer you choose, not any of the linked code in any other language!

 

Think about whether you want your canvas to be separate from the video player, cover the video player, or even cover the entire window.

 

Please use your judgment (and justify your decision in your documentation) as to the precision with which you allow the user to determine the value of an attribute, and whether it is set to an absolute value or modified relative to its current value.

 

When deciding which gesture recognizer to use, please take into account that the points you read in from the mouse are also available for you to process in your own code. You can compute your own information about a gesture while you are accumulating a stroke to send to the recognizer, or after you have accumulated the stroke, or even after you have performed gesture recognition.  For example, you could base your decision about what action to perform on factors such as where a gesture begins or ends, or the direction in which its first or last few points are drawn, in addition to the template match returned by the recognizer.

 

If you decide to use a multistroke recognizer, you can use information that you compute yourself from the points on a stroke, or use a timeout, to determine whether you will allow the user to enter additional strokes before performing recognition. This could be a nice alternative to an explicit trigger to invoke recognition, such as a right mouse click.
